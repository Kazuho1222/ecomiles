# EcoMiles（エコマイルズ）要件定義書

## 1. プロジェクト概要

### 1.1 目的
本プロジェクトは、車などの使用による排気ガスの削減、地球温暖化抑制、環境改善を目的とし、自転車走行、ウォーキング、ランニングなどの環境に優しい移動手段を奨励するウェブアプリケーションを開発する。

### 1.2 背景
地球温暖化や大気汚染などの環境問題が深刻化する中、個人の行動変容を促進するインセンティブシステムが必要とされている。本サービスは、日常的な移動手段の選択に影響を与え、環境負荷の低い移動手段の利用を促進することを目指す。

### 1.3 概要
Stravaと連携し、ユーザーのアクティビティデータを取得。移動距離に応じてポイントを付与。ランキング・バッジ・環境貢献指標として活用する。

## 2. 機能要件

### 2.1 基本機能
1. **Strava連携機能**
   - ユーザーはStravaアカウントとの連携を行うことができる
   - 連携後、自宅を起点としたライド、ウォーキング、ランニングのアクティビティデータを取得する

2. **ポイント付与機能**
   - アクティビティの距離に応じてポイントを付与する
   - 付与レート
   	- 🚶 ウォーキング → 1km = 1pt
   	- 🏃 ランニング → 1km = 1.5pt
   	- 🚴 自転車 → 1km = 0.5pt
   - ポイントの履歴や合計を表示する

3. **可視化機能**
   - 総移動距離
   - 月間/週間アクティビティ
   - CO2削減量の推定表示
   - カレンダー形式の活動ログ

4. **ユーザー管理機能**
   - ユーザー登録、ログイン、ログアウト機能
   - プロフィール管理機能
   - 活動履歴表示機能

5. **ランキング機能**
   - 週間ランキング
   - 月間ランキング
   - 総合ランキング

6. **バッジ・実績機能（ゲーミフィケーション）**
   - 初回達成バッジ（例：初回10km）
   - 継続バッジ（例：7日連続）
   - 距離バッジ（例：100km達成）

### 2.2 追加機能（将来的な拡張）
1. **コミュニティ機能**
   - フォロー機能
   - アクティビティ共有
   - コメント/いいね

2. **チャレンジ機能**
   - 「今月50km達成」などのミッション
   - 個人/グループチャレンジ

3. **ソーシャル共有**
   - SNS投稿機能（Twitter等）

4. **プレミアム機能（将来の収益化）**
   - 詳細分析データ
   - 広告非表示
   - 限定バッジ

## 3. 非機能要件

### 3.1 ユーザビリティ
- レスポンシブデザインによるマルチデバイス対応
- 直感的なユーザーインターフェース
- データの視覚的表現（グラフ・チャート）

### 3.2 パフォーマンス
- ページロード時間：3秒以内
- StravaAPIとの連携処理：5秒以内

### 3.3 セキュリティ
- ユーザー認証情報の安全な管理
- Strava連携用のOAuth認証の適切な実装
- 個人情報保護のためのデータ暗号化
- HTTPS通信

### 3.4 拡張性
- 将来的な機能追加に対応できる設計
- ポイントロジックの柔軟性

## 4. システム構成

### 4.1 開発環境
- フロントエンド：Next.js / React / TypeScript
- バックエンド：Next.js API
- データベース：Supabase（PostgreSQL）
- 認証：Clerk

### 4.2 外部連携
- Strava API
- 広告プラットフォーム（AdSense等）

## 5. 運用モデル

### 5.1 収益モデル
- 主な収益源：広告収益
- 広告表示形式：バナー広告、コンテンツ内広告など
- 将来的にプレミアム課金

### 5.2 コスト構造
- インフラ：無料枠中心（Vercel / Supabase）
- API：Strava無料利用（制限あり）
- ユーザーへの金銭的報酬はなし

### 5.3 KPI
- MAU（月間アクティブユーザー）
- 継続率（リテンション）
- 平均移動距離
- アクティビティ投稿数

## 7. リスクと対策

### 7.1 モチベーション低下リスク
- 対策：バッジ・ランキング・継続ボーナス

### 7.2 API制限リスク
- 対策：キャッシュ・バッチ処理

### 7.3 競合リスク
- 対策：UXと継続性の強化

## 8. 評価指標

1. 6ヶ月以内にMAU1000人
2. 継続率60%以上
3. ユーザー平均週2回以上のアクティビティ

## 9. 補足（設計思想）

本サービスは「報酬」ではなく以下を価値とする：
- 達成感（バッジ）
- 可視化（データ）
- 競争（ランキング）
- 習慣化（継続記録）
これにより、低コストかつ持続可能なサービス運用を実現する。
